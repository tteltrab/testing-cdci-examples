Deployment Problems

Who in here has done a manual deploy before?

All problems surrounding it being a very manual proces:

  • Error prone
  • Slow
  • Tedious
  • Takes away from development time.

Deployment Solutions

Continuous deployment is, in short, automation of the software delivery process.

The idea is that every time you deliver code, a set of tools and infrastructure work together automatically to build and push your code to production, without any need for human intervention.

There are many such tools, but the tool we'll be looking at today is TravisCI.


CD Benefits

  • Less error prone
  • Save developer time
  • Ease of development
  • Reduced lead time (time to delivery)
    • Earlier ROI, cheaper to get things to production
    • Earlier user feedback

Activity: Travis CI Setup

We're going to use my testing demo repo as a starting point.

  • Create a personal access token. Github Token Settings
  • Copy that token, and then go to the Travis CI build settings.
  • Create an environment variable with the key GH_TOKEN whose value is the token you just copied. This allows Travis to do the push to the branch to the gh-pages branch every time the build passes.
  • On line 14 of the deploy script change the email to your github email.
